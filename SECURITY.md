# Security Policy

ArIEC103 is a protocol testing and evidence tool. Please treat relay captures, serial traces, screenshots, project names, IP addressing, and utility/customer data as potentially sensitive.

## Reporting a vulnerability

Use GitHub private vulnerability reporting if it is enabled for the repository. If it is not available, open a minimal public issue that describes the problem class without attaching sensitive traces or customer data.

Do not publish real relay captures, customer names, utility names, site names, credentials, serial numbers, private IP schemes, or project documents in public issues.

## Supported scope

The public repository supports the current main branch and the latest tagged release.

## Responsible disclosure expectation

Please provide enough technical detail to reproduce the issue using simulated or sanitized data. Maintainers may ask for a redacted trace or a synthetic reproduction case.
