Workflow

Built around how engineers actually test relay communication.

The primary path is active master testing: configure COM and IEC addresses, start the session, run GI and background polling, drain Class 1 only when appropriate, then export an evidence report. Offline decode remains available, but it is not the product ceiling.

M

Master Control

Reset FCB, optional clock sync, General Interrogation, Class 2 polling, bounded Class 1 drain, timeout recovery, and stop-safe serial handling.

V

Value Viewer

Shows the latest relay values using user mapping when available, while retaining raw FUN/INF evidence when a point is unmapped.

E

Relay Event Log

Records state changes and spontaneous relay events using the relay timestamp from IEC-103 ASDU time fields, not PC arrival time.

Architecture

Focused on one real connection, deep evidence, and predictable memory use.

The stack separates active master logic, FT1.2 decoding, user mapping, operator output, frame trace transparency, and bounded UI buffers so the tool stays responsive during high-volume polling.

Field truth

Raw hex stays visible, but it is not the language of the main workflow.

Operator view first. Users see signal, state, relay time, quality, source, finding, and recommended action before touching raw bytes.

Protocol transparency remains. Expert users can inspect raw TX/RX frames, checksum, PRM/FCB/FCV/ACD/DFC, ASDU, COT, FUN, INF, DPI, and relay timestamp.

Mapping belongs to the user. ArIEC103 does not ship fake vendor profiles. Signal names come from project-specific mapping profiles; raw FUN/INF remains the source of truth.
